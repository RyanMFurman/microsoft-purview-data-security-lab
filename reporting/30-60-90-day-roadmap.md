# 30/60/90-Day Roadmap

**Evidence label: DESIGNED FOR PURVIEW**

## First 30 days — Discover, own, and pilot safely

| Action | Owner | Dependency | Evidence/KPI |
|---|---|---|---|
| Confirm authorization, licenses, roles, workloads, and owners | Program Lead | Tenant sponsorship | Approved scope/capability matrix |
| Inventory PHI repositories, links, guests, and broad groups | SharePoint + Data Owners | Read access | Repository ownership and access-review coverage |
| Remediate Critical anonymous/stale access | Site Owners + IAM | Change approval | Critical findings closed |
| Create and publish four labels to P1 pilot | Compliance Admin | Licensing/roles | Label visibility/order tests |
| Test E1 PHI encryption | Compliance Admin + Data Owner | Approved identities | Authorized/unauthorized access results |
| Establish baseline metrics and support path | Security Operations | Reporting ownership | Baseline dashboard and escalation path |

## Days 31–60 — Tune controls and validate response

| Action | Owner | Dependency | Evidence/KPI |
|---|---|---|---|
| Run PHI DLP simulation in SharePoint/OneDrive | Compliance Admin | SIT and pilot locations | Match and false-positive rate |
| Add policy tips and test D1 overrides | Security + Data Owner | User communications | Override rate and justification quality |
| Execute investigation tabletop | Data Security Analyst | Audit/evidence roles | Evidence completeness and containment time |
| Approve lifecycle schedules and test R1 | Records + Privacy + Owners | Legal validation | Hold/disposition/deletion tests |
| Validate sanitized configuration exports | Compliance Engineering | Read-only export | Validator results and drift findings |
| Prepare Copilot users and repositories under G1 | Copilot Program Owner | Access reviews | Five users trained; repositories approved |

## Days 61–90 — Enforce narrowly and decide expansion

| Action | Owner | Dependency | Evidence/KPI |
|---|---|---|---|
| Enforce tuned DLP for approved pilot | Compliance Admin | Simulation approval | Block/override outcomes and support impact |
| Apply no-override blocking to proven anonymous/repeat cases | Security + Data Owner | Change approval | Prevented high-risk events |
| Launch five-user Copilot pilot | Copilot Program Owner | G1 gate complete | Usage, support, DLP, and audit metrics |
| Review DSPM insights/recommendations where licensed | Data Security | Telemetry/propagation | Accepted/rejected recommendations with rationale |
| Report residual risk and control effectiveness | Program Lead | Valid metrics | Executive decision package |
| Approve, defer, or stop expansion | Executive Risk Owner | Residual-risk acceptance | Signed expansion decision |

## 90-day success definition

- Every pilot repository has an owner and completed access review.
- Critical oversharing findings are closed or formally accepted.
- PHI labels, encryption, DLP, audit, and investigation tests meet approved criteria.
- R1 AI evaluation lifecycle has approved ownership and exception handling.
- Copilot expansion is evidence-gated rather than schedule-driven.

## Known dependencies

- Feature-level licensing and product availability
- Tenant authorization and least-privileged roles
- Data-owner and Legal/Privacy/Records participation
- Audit and telemetry propagation time
- Office, SharePoint, OneDrive, and client prerequisites
- User training and support capacity
